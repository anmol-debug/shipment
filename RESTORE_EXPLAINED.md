### Restore to Version 1: How It Works Internally

#### Step 1: Read Source Snapshot
```sql
SELECT snapshot_data FROM shipment_versions
WHERE shipment_id = '550e8...000' AND version_no = 1;
```

**Returns:**
```json
{
  "title": "Ship LA",
  "status": "pending",
  "container": "ABCD1234567"
}
```

**Disk Read:** Fetches data from block 0x1000

---

#### Step 2: Create New Audit Event (Version 4)
```sql
INSERT INTO shipment_audit_events (
  shipment_id, version_no, event_type, actor_id, actor_name,
  timestamp, reason, field_changes, metadata
) VALUES (
  '550e8...000', 4, 'restored', '550e8...001', 'TestUser',
  NOW(), 'Revert to original container',
  '{"source_version_no": 1}',
  '{"restored_from": 1}'
);
```

---

#### Step 3: Copy Snapshot Data to New Version
```sql
INSERT INTO shipment_versions (
  shipment_id, version_no, snapshot_data, created_by
) VALUES (
  '550e8...000', 4,
  '{"title": "Ship LA", "status": "pending", "container": "ABCD1234567"}',
  '550e8...001'
);
```

**Key Point: COPYING, NOT POINTING**
- Version 1 data at block 0x1000: `{"container": "ABCD1234567"}`
- Version 4 data at block 0x4000: `{"container": "ABCD1234567"}` â† **NEW COPY**
- Both contain IDENTICAL data but stored in DIFFERENT locations

---

#### Step 4: Update Current State
```sql
UPDATE shipment_requests
SET
  title = 'Ship LA',
  status = 'pending',
  container = 'ABCD1234567',
  updated_at = NOW()
WHERE id = '550e8...000';
```

## Summary

### What Gets Stored:
| Version | Event Type    | Snapshot Data              | Storage   |
|---------|---------------|----------------------------|-----------|
| 1       | created       | Original data              | New copy  |
| 2       | updated       | Modified data              | New copy  |
| 3       | status_change | Modified data              | New copy  |
| 4       | restored      | **Copy of version 1 data** | New copy  |

### Key Points:
1. **Each version has its own complete snapshot** (copying)
2. **No pointers** - each snapshot is self-contained
3. **History never changes** - versions 1-3 remain exactly as they were
4. **Restore = new version** - version 4 is created, not version 1 modified
5. **All metadata recorded** - who, when, why, from which version

### Benefits:
- âœ… Simple queries: `SELECT snapshot_data WHERE version_no = 4`
- âœ… Fast reads: No joins, no pointer chasing
- âœ… Reliable: Each version independent
- âœ… Easy to understand: What you see is what you get

### Trade-offs:
- âŒ Storage overhead: Data duplicated (but storage is cheap)
- âŒ Larger database size: But manageable for most use cases
